## 2 способа аутентификации SSH
* по паролю - дефолтный способ, но не такой надежный, потому что пароль может и утечь
* по ключу SSH - более безопасный. Есть такая тема, работает как есть закрытый(как замок
) и открытый ключ(как личный открытый ключ)

1. На компуктере создаете пару ключей (открытый и закрытый ключи)
2. Открытый ключ мы передаем на сервер, а закрытый хранится на нашем компе.
3. Когда мы подключаемся к серверу, SSH использует закрытый ключ, 
чтобы доказать, что то действительно ты

## Для генерации SSH ключей
Для этого используется зачастую утилита ssh-keygen

```bash
ssh-keygen -t ed25519
```

алгоритм Ed25519 - Это криптографический алгоритм для генерации SSH-ключей. Он о
беспечивает высокую безопасность при сравнительно короткой длине ключа и считается одним 
из самых надёжных и быстрых
----

при создании будет предложено задать пароль для ключа, как доп перестраховка, но можно забить 
и скинпуть этот этап

По умолчанию ssh-keygen сохранит ключи в `~/.ssh`

если нужно будет создать несколько пар, можно воспользоваться флагом `-f` для того, чтобы указать путь
чтобы потом указать путь к нужному, надо воспользоваться `-i`

## ssh-agent

Как работает ssh-agent:
Запуск. Ssh-agent запускается один раз и остаётся в фоновом режиме, создавая уникальный процесс. В большинстве Unix-подобных систем его можно запустить командой eval "$(ssh-agent -s)".
Добавление ключей. Используйте команду ssh-add, чтобы добавлять приватные SSH-ключи в ssh-agent и использовать их для аутентификации. Если ключ защищён паролем, ssh-add попросит ввести его только один раз.
Аутентификация. Когда вы подключаетесь к серверу, SSH-клиент обращается к ssh-agent для получения ключа. Ssh-agent предоставляет данные, необходимые для аутентификации, позволяя пользователю войти без повторного ввода пароля к ключу.

1️⃣ Откройте терминал в Linux, macOS или запустите PowerShell в Windows 10 и 11 и выполните команду:
